What's on the cards

A whole wiki, in one small app.

Everything a personal knowledge base needs, and nothing that needs a cluster to run.

Write

An editor that stays out of the way

Markdown with a rich-text surface, live word count, and autosave. Footnotes, uploaded images, and drafts & stubs for half-formed thoughts.

Link

Backlinks, red links, and stubs

Type [[ to link. Every entry lists what references it. Unwritten cards show as red links; create one inline and it becomes a followable stub.

Find

Real full-text search

SQLite FTS5 with ranked results and highlighted excerpts across titles and bodies. Plus an A–Z index and a random card button.

Keep

Every save is logged

A per-entry changelog records who changed what, and when. Soft edit-locks keep two people from clobbering each other. Backup is copying one file.

Share

Publish and syndicate

Atom and JSON feeds, a weekly email digest, Slack announcements, Mailchimp sync, and signed webhooks — or flip the whole site private.

Invite

Passwordless, multi-user if you want

Login is a magic link — no passwords stored, ever. Add co-writers with roles, and an audit log records who did what.

File under: installation

On a server in about ten minutes.

Ubuntu or Debian, one script. It installs nginx, gunicorn, and certbot, creates a service user, writes systemd units, and gets a TLS certificate.

  1. Get the code onto your server.

    sudo git clone https://github.com/rudzki/indexcards.git /srv/indexcards

  2. Run the setup script as root.

    cd /srv/indexcards/deploy && sudo ./setup.sh

  3. Open your domain and create the first admin account in the browser. Write your first card.

    Prefer to try it locally first?

    python -m venv venv && source venv/bin/activate
pip install -r requirements.txt && python run.py

The technical details.

Deliberately boring infrastructure. No Redis, no queue, no node_modules in production.

StackPython · Flask 3.1 · Flask-SQLAlchemy · server-rendered Jinja2 — vanilla JS, no SPA framework
DatabaseSQLite, single file, with FTS5 for search — backup = copy one file
EditorProseMirror, vendored and bundled at build time — no CDN at runtime
Markdownmistune renderer · bleach sanitizer with a strict tag allowlist
AuthPasswordless email magic links · CSRF everywhere · per-route rate limits
EmailAny SMTP server, set in the admin panel — prints to console in dev
APIRead-only JSON API /api/v1/entries, paginated and rate-limited
TestedA pytest suite runs against in-memory SQLite — entries, auth, search, feeds, integrations
Deploymentgunicorn behind nginx · systemd service + digest timer · Let's Encrypt via certbot
FootprintRuns comfortably on 512 MB RAM
