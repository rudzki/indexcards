Self-hosted · Open source · SQLite

Your knowledge base,
on your own server.

Index Cards is a lightweight wiki for personal or team use. Write entries in Markdown, link them together, track backlinks, and send a weekly digest to subscribers. One SQLite file. No external services required.

Get started Read the docs
What's included

Everything a knowledge base needs

No plugins, no subscriptions, no third-party accounts required to get started.

✎
Rich-text editor
ProseMirror editor with toolbar for headings, lists, code blocks, links, blockquotes, and footnotes. Markdown source always accessible.
🔗
Internal links & backlinks
Link entries with [[Title]] syntax. Every entry shows what links to it. Missing targets flagged with a dashed underline.
🔍
Full-text search
SQLite FTS5 search with highlighted excerpts and prefix matching. Kept in sync via database triggers — no indexer process needed.
📧
Weekly email digest
Double opt-in subscriptions and a weekly digest of new (and optionally edited) entries. Run via cron; falls back to console when SMTP is absent.
👤
Multi-user with roles
Four roles: Viewer, Author, Editor, Admin. Passwordless magic-link login. Invite-only, public, or email-domain-restricted registration.
📄
Drafts & changelogs
Entries live in draft until published. Each save can include a changelog message. Full edit history visible on every entry.
🔁
Import & export
Import from JSON or WordPress XML. Export all entries as a Markdown ZIP or JSON. No lock-in.
🔔
Integrations
Slack announcements, signed outgoing webhooks, and Mailchimp audience sync — all configurable from the admin UI, no code required.
🔒
Private mode
Set visibility to "registered" and the entire site requires a login. Useful for internal team wikis.
Under the hood

Boring technology, deliberately

Standard Python web stack. Nothing exotic to install or operate.

Backend
Python 3.10+, Flask 3
Database
SQLite via Flask-SQLAlchemy
Migrations
Auto-applied at startup, no CLI step
Markdown
Mistune 3 + Bleach sanitisation
Editor
ProseMirror (vendored, no build step)
Auth
Magic-link tokens, Flask-Login
Security
CSRF (Flask-WTF), rate limiting
Production
Gunicorn + nginx or Caddy
Getting started

Up in five minutes

Clone, install, run. The setup wizard creates the first admin account.

01
Clone & install
Clone the repo, create a virtualenv, and pip install -r requirements.txt. No Node.js, no build step.
02
Set three env vars
SECRET_KEY, SITE_URL, and optionally DATABASE_URL. Everything else is configured in the admin UI.
03
Run
gunicorn run:app in production. The database is created automatically. Visit /setup to create the admin account.
04
Configure in the UI
SMTP, integrations, registration settings, digest schedule — all editable from Dashboard → Settings with no server restart needed.
Installation

One command to install

Requires Python 3.10+. SQLite is included in the standard library.

# Clone and install
$ git clone https://github.com/rudzki/indexcards
$ cd indexcards
$ python -m venv venv && source venv/bin/activate
$ pip install -r requirements.txt

# Set required environment variables
$ export SECRET_KEY="$(python -c 'import secrets; print(secrets.token_hex(32))')"
$ export SITE_URL="https://example.com"

# Run (development)
$ python run.py

# Run (production)
$ gunicorn run:app --workers 2 --bind 127.0.0.1:8000
No database setup
SQLite database is created automatically at instance/indexcards.db. Schema migrations run on every startup — no flask db upgrade step.
First run
Visit the app and you'll be redirected to /setup to create the admin account and set the site title.
SMTP is optional
Without SMTP, login links and digest emails are printed to the console. Configure SMTP from the admin UI after setup.
Weekly digest
Add flask send-digest to a cron job to send the weekly digest to subscribers. Runs independently of the web process.
